# 스프린트11

이번에는 개인적으로 `웹소켓` 에 대해서 개인적으로 학습을 했습니다.

> www.messagoom.online

## 주요 변경점

1. koa 라이브러리 사용
2. drizzle orm 사용
3. gateway, chat, game 을 통한 마이크로서비스 패턴? 흉내

나름 가벼운 환경을 만든다고 `koa` 를 사용했는데, 이것저것 추가하고 보니 `fastify` 가 더 가볍고 빠르다는 것을 알게되었습니다.

단순히 `express` 의 콜백 기반이 async/await 기본 형태인 것과. declare 없이 특정 추가 정보들을 넣을 수 있다..? 는 점은 편했습니다.

기존 판다마켓에서 웹소켓 적용이 생각보다 적었어서, 개념은 이해가 되는데 어떻게 활용할지 감이 잘 안 왔었습니다.

그래서 웹소켓을 주구장창 쓰는 실시간 동기화 되는 온라인 게임을 만들어서 웹소켓을 가능한 많이 써보려고 애썼습니다.
